作者认为：开发中要尽量避免为滚动容器设置padding-bottom值，除了样式表现不一致外，还有导致scrollHeight值不一样，这往往会给开发带来难以察觉的麻烦。

了解overflow-x和overflow-y

如果overflow-x和overflow-y属性中的一个值被设置为visible而另一个值设置为scroll，auto或是hidden，则visible的表现会如同auto。

overflow与滚动条

(1)在PC端，不论是什么浏览器，默认滚动条均来自"html"标签，而不是"body"标签

在PC端，窗体滚动高度可以使用document.documentElement.scrollTop获取，但是在移动端，可能要使用document.body.scrollTop获取。

(2)滚动条会占据容器的可用宽度或是高度。(17px)

但是在移动端不会有这个问题，因为移动端滚动条一般都是悬浮模式。

获取浏览器的滚动栏宽度

滚动栏占据宽度特性最大的问题是页面加载的时候水平居中的布局可能会发生晃动，因为窗体本来是没有滚动条的。

页面不晃动的技巧：

            html{
                overflow-y:scroll;/* for IE8*/
            }
            :root{
                overflow-y: auto;
                overflow-x:hidden;
            }
            :root body{
                position: absolute;
            }
            body{
                width: 100vw;
                overflow: hidden;
            }
